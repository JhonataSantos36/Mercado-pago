Información obligatoria para cobrar

Necesitás recolectar la información con la que desde tu servidor vas a invocar nuestros servicios. Es obligatorio que al momento de hacer el pago nos envíes los siguientes datos:

  • Monto que deseas cobrar.
  • Mail del comprador: Sólo por si algo extraño sucediera durante o luego del pago. Esto es una protección tanto para tu usuario como para tu servicio posventa.
  • Medio de pago: La marca de la tarjeta, la entidad de cobro en efectivo (ya sea un cajero automático o por ventanilla). Cada medio de pago tiene sus particularidades: tiempo de acreditación para medios de pago offline; longitud del número de tarjeta, del código de seguridad, entre otros para medios de pago de tarjetas.
  • Banco (sólo para tarjetas): Es el emisor de la tarjeta. Con quienes establecemos planes de cuotas sin interes para que tus usuarios paguen de manera más cómoda.
  • Cuotas (sólo para tarjetas): La cantidad de cuotas seleccionadas por tu usuario. Nosotros nos encargamos de saber si el pago debe ser procesado con o sin interés dependiendo del medio de pago y el banco de la tarjeta.
  • Token (sólo para tarjetas): Necesitamos el número, el nombre, la fecha de expiración y el código de seguridad de la tarjeta de tu usuario. En algunos países también es necesario que nos indiques su tipo y número de documento. MercadoPago tokeniza estos datos para proteger la información sensible de la tarjeta.

¿Qué necesitás para empezar?

  • Tener o crear una cuenta de MercadoPago dónde recibirás el dinero.
  • Crear una aplicación.
  • Configurar tus credenciales.

Modo sandbox y productivo

Con el Checkout personalizado puedes recibir pagos en modo sandbox o productivo. La diferencia para procesar en uno u otro modo está en el tipo de credenciales que uses para autenticarte en la API.
Desarrolla tu aplicación usando tus credenciales para modo sandbox. Cuando esté lista para recibir pagos productivos, deberás pasar un proceso de homologación y reemplazar las credenciales por las de producción. No necesitas cambiar otra cosa de tu código. De esta forma puedes realizar pagos de prueba mientras tus clientes están procesando pagos productivos con sólo utilizar las claves correspondientes.
El entorno de sandbox es una réplica exacta de producción, diseñado con el objetivo de facilitar las pruebas durante la integración. El uso de este modo no genera ningún tipo de cargo real. Además de las credenciales correspondientes, en modo sandbox debes utilizar las tarjetas de prueba que te brindamos.

Usá tus credenciales

Cada aplicación tiene dos pares de claves para conectarse con la API de MercadoPago. Estas claves las encuentras en la sección credenciales de tu cuenta.
Las claves de modo sandbox te permiten realizar pagos de prueba y las de modo producción, pagos reales.
Además de los modos sandbox y producción, una de las claves es pública y la otra privada.
Las claves públicas identifican tu cuenta de MercadoPago y no son secretas. Se utilizan en el frontend.
Con las claves públicas puedes acceder a los siguientes recursos mediante nuestra SDK:

  • Medios de pago (payment_methods)
  • Bancos (card_issuers)
  • Cuotas (installments)
  • Tipos de documento (identification_types)
  • Tokenización de datos (card_tokens)
Las claves privadas deben ser mantenidas confidencialmente en tus servidores de backend y nunca deben ser publicadas.
Estas claves te permiten acceder a los siguientes recursos:
  • Pagos (payments)
  • Devoluciones (refunds)
  • Clientes (customers)
  • Tarjetas (cards)

Recibí notificaciones de pagos

Cada vez que recibes un pago o se modifique uno que te hicieron, te podemos avisar mediante Notificaciones de Pagos que se envían desde los servidores de Mercadopago a los tuyos. Esto te permitirá conocer el estado del pago, administrar tu stock o saber quién te está pagando.
