3. Start the checkout from your app

Try it out with our data:
Use this code with our data to make a quick trial of the full experience.
When you want to go into production, you'll just have to change the data and use yours.

To start our checkout you just need:

  • Public key: it's an identifier that groups your account, app and its configurations.
    • Create your credentials.
    • Configure your app.
  • Payment preference identifier: it's a representation of what you want to be paid for. It includes this attributes:
    • Data and amount.
    • Accepted payment methods.
    • Payer information (the email is mandatory for security reasons).
  • Optional: you can change our checkout colors using the DecorationPreference class:
    • Create an instance: DecorationPreference decorationPreference = new DecorationPreference();
    • Configure a base color: decorationPreference.setBaseColor(color);
    • You can enable a dark font: decorationPreference.enableDarkFont();
    • Add it to the checkout's setters!

                     
// Executed method when clicking the button
public void submit(View view) {

  // Iniciar el checkout de MercadoPago

    .setActivity(this)

    .setCheckoutPreferenceId("150216849-ceed1ee4-8ab9-4449-869f-f4a8565d386f")
    .startCheckoutActivity();

}

4. Wait for the payment result

The SDK will always return a payment result

If an insurmountable error occurs or the user cancels the operation, it will return an exception so you know what happened.

These are the most important attributes of the payment object:

  • id: payment identifier.
  • status: payment status.
  • payment_method_id: an identifier of the payment method that your user selected.
  • payment_type_id: selected payment method type.
  • card: object that identifies your user's card.
  • issuer_id: an identifier of your user's selected card issuer.
  • installments: amount of installments selected.

Get to know more about the Payment object here. 

                     
// Wait for the checkout result
@Override
protected void onActivityResult(int requestCode, int resultCode,
Intent data) {

  if (requestCode == MercadoPago.CHECKOUT_REQUEST_CODE) {
    if (resultCode == RESULT_OK && data != null) {

      // Done! The payment was already processed by MercadoPago
      Payment payment = JsonUtil.getInstance()
        .fromJson(data.getStringExtra("payment"), Payment.class);
      TextView results = (TextView) findViewById(R.id.mp_results);

      if (payment != null) {
          results.setText("PaymentID: " + payment.getId() +
          " - PaymentStatus: " + payment.getStatus());
      } else {
          results.setText("The user did not complete the payment.");
      }

    } else {
      if ((data != null) && (data.hasExtra("mpException"))) {
          MPException mpException = JsonUtil.getInstance()
            .fromJson(data.getStringExtra("mpException"), MPException.class);
          // Handle errors.
      }
    }
  }
}

6. Go into production

First:

  • Register a MercadoPago account to receive money.
  • Create an application.
  • Configure your credentials.

From your server:
Just create a payment preference and return our services response. Do it from your server, because you will have to sign it with your private key. By doing this we protect the payer and also your own seller user. How to create a preference in your server.

Complete preferences perform much better.
Send us all the information you can, and you will offer such an amazing experience for your buyers that your accredited payments will raise!

From your app:
We offer you a class called MerchantServer that connects with your server. The createPreference method makes a POST and sends as the message's body the map you've defined (preferenceMap). Set your base URL (https://api.tunombre.com) and URI (/create_preference) where you expect to receive the information in order to create a preference.
MerchantServer will turn your service's response (the same from MercadoPago services) into a CheckoutPreference object, which ID is the entry point to our checkout. 

                     
// Heads up! It's a MercadoPago callback
import com.mercadopago.callbacks.Callback;

public void submit(View view) {

  // Create a map with the purchase data and your client's email.
  Map<String, Object> preferenceMap = new HashMap<>();
  preferenceMap.put("item_id", "1");
  preferenceMap.put("amount", new BigDecimal(10));
  preferenceMap.put("currency_id", "ARS");
  preferenceMap.put("payer_email", "client_email@gmail.com");

  // Send the information to your server
  MerchantServer.createPreference(this, "https://api.yourname.com",
    "/create_preference", preferenceMap, new Callback<CheckoutPreference>() {

      @Override
      public void success(CheckoutPreference checkoutPreference) {
          // Preference correctly created
          // Start MercadoPago Checkout.
          new MercadoPago.StartActivityBuilder()
            .setActivity(mActivity)
            .setPublicKey("TEST-ad365c37-8012-4014-84f5-6c895b3f8e0a")
            .setCheckoutPreferenceId(checkoutPreference.getId())
            .startCheckoutActivity();
      }

      @Override
      public void failure(ApiException error) {
        // Oops, something went wrong
      }
  });
}
